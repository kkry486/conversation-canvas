> 版本：v0.3.0
> 日期：2026年6月18日

## 1. 项目概述

对话画布是一个基于节点图的 AI 对话系统，借鉴 ComfyUI 的交互方式，将传统的线性对话界面改造为可视化的节点编辑器。用户可以在画布上创建对话节点、连接连线、管理多条对话分支。

### 已实现功能

| 功能 | 状态 | 说明 |
|------|------|------|
| LiteGraph.js 画布集成 | ✅ | 深色主题，支持缩放、拖拽、连线 |
| 模型配置节点 | ✅ | 下拉选择模型、API Key、Base URL、System Prompt |
| 对话输入节点 | ✅ | 文本输入 + 发送按钮 |
| AI 回复节点 | ✅ | 流式显示回复、状态指示、自动扩展高度、滚轮滚动 |
| 自动连线 | ✅ | 发送后自动创建回复节点和新输入节点并连线 |
| 分叉对话 | ✅ | 从任意节点创建分支，各自独立回溯祖先路径 |
| 对话历史回溯 | ✅ | 从当前节点沿图路径回溯祖先，自动组装上下文 |
| System Prompt | ✅ | 模型配置节点支持系统提示词，作为首条 system 消息 |
| 回复自动扩展 | ✅ | 回复节点根据内容长度自动扩展高度（80~400px） |
| 回复滚轮滚动 | ✅ | 鼠标滚轮在回复节点上滚动内容，其他地方缩放画布 |
| 文本复制 | ✅ | 「选择复制」按钮弹出文本选择框，可自由选中复制 |
| DeepSeek API 接入 | ✅ | 支持 deepseek-v4-pro / deepseek-v4-flash |
| SSE 流式响应 | ✅ | 后端代理转发，前端逐字渲染 |
| 手动创建节点 | ✅ | 右键/双击搜索框添加节点，自动绑定发送回调 |

## 4. 架构设计

### 4.1 数据流

```
┌──────────────┐   output[0]   ┌──────────────┐   output[0]   ┌──────────────┐
│  模型配置节点  │ ───────────→ │  对话输入节点  │ ───────────→ │  AI 回复节点   │
│  ModelConfig  │  (配置对象)    │  PromptNode  │  (消息文本)    │ ResponseNode │
└──────────────┘               └──────────────┘               └──────────────┘
                                   ↑ input[0] 配置                  │
                                   │ input[1] 上下文                 │ output[0] 内容
                                   ↓                                ↓
                            ┌──────────────┐               ┌──────────────┐
                            │  新输入节点    │ ←─────────────│  （自动创建）  │
                            └──────────────┘
```

### 4.2 节点端口定义

**模型配置节点**
- `output[0] 配置 (object)`：model、apiKey、baseUrl、systemPrompt

**对话输入节点**
- `input[0] 配置 (object)`：模型配置
- `input[1] 上下文 (string)`：上一个回复
- `output[0] 消息 (string)`：用户输入
- `output[1] 配置 (object)`：透传配置

**AI 回复节点**
- `input[0] 回复 (string)`：上游消息
- `output[0] 内容 (string)`：AI 回复全文

### 4.3 对话历史回溯机制

每次发送消息时，从当前 PromptNode 沿图路径向上回溯，提取沿途的用户消息和 AI 回复，组装为 OpenAI messages 数组。分叉场景下各分支独立回溯，互不干扰。

### 4.4 回复节点滚轮滚动

LiteGraph 默认将滚轮事件用于画布缩放。通过替换 canvas 的 `mousewheel` 事件监听器，在鼠标位于回复节点上时拦截滚轮事件用于内容滚动，其他区域正常缩放。使用 `node._scrollable` 标记区分可滚动节点。

## 5. 后端 API

### `POST /api/chat`

**请求体**：

```json
{
  "model": "deepseek-v4-pro",
  "apiKey": "sk-xxx",
  "baseUrl": "https://api.deepseek.com",
  "systemPrompt": "你是一个助手",
  "message": "用户当前消息",
  "history": [
    {"role": "user", "content": "之前的用户消息"},
    {"role": "assistant", "content": "之前的AI回复"}
  ]
}
```

**响应**：SSE 流式返回（`text/event-stream`）

**支持的模型**：

| 模型 ID | 说明 |
|---------|------|
| `deepseek-v4-pro` | DeepSeek V4 Pro，深度推理 |
| `deepseek-v4-flash` | DeepSeek V4 Flash，快速响应 |
| `mimo` | MiMo 模型 |

## 6. 关键技术决策

### 6.1 LiteGraph 原版 vs ComfyUI fork

npm 上的 `litegraph.js@0.7.18` 是原版，没有 `addDOMWidget`（ComfyUI fork 才有）。使用原版的 `addWidget` 和 `addCustomWidget` API。

### 6.2 回复节点多行渲染

使用 `addCustomWidget` 自定义绘制，核心函数在 `widgets.js` 的 `drawScrollableText`：

- Canvas 上逐字符换行绘制文本
- `computeSize` 动态计算节点高度（80~400px）
- 流式输出时自动滚动到底部 + 闪烁光标
- 滚轮滚动通过 canvas 级别的 `mousewheel` 事件拦截实现

### 6.3 文本复制方案

Canvas 绘制的文字无法直接选中。采用「选择复制」按钮方案：点击后弹出 DOM textarea 浮层，内容自动全选，用户可自由选择复制。

### 6.4 Canvas 初始化

```javascript
canvasElement.width = rect.width;   // 必须设置实际像素尺寸
canvasElement.height = rect.height;
canvas.resize();
```

### 6.5 SSR 禁用

```javascript
export const ssr = false;
export const csr = true;
```

### 6.6 属性名冲突

LiteGraph 会覆盖节点上以 `_` 开头的属性。回调使用 `sendCallback` 而非 `_onSend`。
