## 3. Agent 执行中禁用发送

**评分：⚠️ 有问题但可修**

**改动位置**：[+page.svelte:28](big/src/routes/+page.svelte#L28), [第 201 行](big/src/routes/+page.svelte#L201), [第 211 行](big/src/routes/+page.svelte#L211)

**实现方式**：
```javascript
// 声明状态
let agentRunning = $state(false);

// 发送前锁定
if (agentMode) agentRunning = true;

// 完成后解锁
try {
  const result = await sendFn(...);
} finally {
  if (agentMode) agentRunning = false;
}
```

**问题**：
1. ❌ `agentRunning` 状态变量虽然正确设置，但**没有绑定到任何 UI 元素的 disabled 属性**
2. ❌ 模板中没有发送按钮（发送由 LiteGraph 画布内部管理）
3. ❌ `agentRunning` 的变化无法影响 LiteGraph 创建的发送按钮

**后果**：
- 用户可以在 Agent 执行期间重复点击发送按钮
- 可能导致多个并发 Agent 任务

**修复建议**：
- 方案 A：在 `bindSend` 回调开头检查并 return（如果正在执行）
  ```javascript
  promptNode.sendCallback = async () => {
    if (agentRunning) return;  // 添加这行
    // ...
  }
  ```
- 方案 B：禁用 LiteGraph 画布的交互（通过 `graphData.canvas.allow_interaction = false`）

**新 bug**：是 — 并发发送风险

**遗漏**：审计报告中的"发送锁定"需求未完全实现
