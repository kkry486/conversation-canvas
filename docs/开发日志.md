# 对话画布 开发日志

> 项目：conversation-canvas
> 仓库：https://github.com/kkry486/conversation-canvas
> 开发周期：2026年6月18日 — 6月19日
> 开发方式：AI 辅助开发（Claude Code）

---

## 第一阶段：项目规划与初始化（6月18日）

### 1.1 需求来源

用户提出将 AI 对话系统做成类似 ComfyUI 的节点图画布效果。

### 1.2 技术选型

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 前端框架 | SvelteKit | 前后端一体，开发效率高 |
| 节点编辑器 | LiteGraph.js（npm 原版） | ComfyUI 同款引擎 |
| 后端 | SvelteKit API Routes | 无需额外服务 |
| AI API | OpenAI 兼容格式 | DeepSeek V4 原生支持 |

### 1.3 项目初始化

手动创建 SvelteKit 项目，14 个源文件，覆盖前端画布、节点定义、后端 API、全局样式。

## 第四阶段：Agent 化（6月18日晚 — 6月19日）

### 4.1 Agent 工具模块

创建 `src/lib/tools/index.js`，实现：

- **5 个工具定义**：read_file、write_file、list_directory、execute_command、search_in_files
- **工具执行器**：每个工具的实际执行逻辑，带路径安全检查
- **Agent 循环引擎**：`runAgentLoop()` 实现 LLM → tool_use → 执行 → 回传 → 继续的循环，最多 10 轮

### 4.2 Agent API 路由

创建 `src/routes/api/agent/+server.js`：

- 接收 chat 请求，启动 Agent 循环
- 通过 SSE 流式推送每一步（thinking、text_delta、tool_call、tool_result、done）
- 前端实时接收事件并创建对应节点

### 4.3 Agent 节点类型

新增 4 个节点类型：

| 节点 | 类型名 | 颜色 | 功能 |
|------|--------|------|------|
| 🧠 思考 | Agent/思考 | 紫色 | LLM 推理过程 |
| 🔧 工具调用 | Agent/工具调用 | 蓝色 | 调用的工具名 + 参数 |
| 📋 工具结果 | Agent/工具结果 | 绿/红 | 工具执行返回值 |
| 📄 文件产物 | Agent/文件产物 | 蓝色 | 创建的文件路径 + 打开按钮 |

### 4.4 文件产物节点

自定义绘制按钮（不用 LiteGraph 内置按钮），解决文字不居中问题：
- **▶ 打开文件**：调用系统默认程序打开文件
- **📂 打开文件夹**：在资源管理器中打开并选中文件
- 后端 `/api/open` 路由支持 `mode: 'file'` 和 `mode: 'folder'`

### 4.5 Agent 模式切换

工具栏新增 **🤖 Agent** 开关按钮，紫色高亮。点击切换对话/Agent 模式。

## 第五阶段：Tauri 启动器（6月19日）

### 5.1 Rust 环境搭建

- 安装 Rust 1.96.0（通过代理加速下载）
- 安装 create-tauri-app 4.7.3
- 安装 tauri-cli 2.11.3

### 5.2 Tauri 项目集成

在现有 SvelteKit 项目中集成 Tauri v2：

```
src-tauri/
├── Cargo.toml          # Rust 依赖：tauri、reqwest、tokio、serde_json
├── tauri.conf.json     # Tauri 配置：窗口尺寸、构建命令
├── build.rs            # 构建脚本
├── capabilities/main.json  # 权限配置
├── icons/icon.ico      # 最小占位图标
└── src/
    ├── main.rs         # Windows 子系统入口
    └── lib.rs          # 核心逻辑：chat、agent、open_file 命令
```

### 5.3 Rust 后端 API

将 Node.js 后端全部迁移到 Rust：

| 原 Node.js API | Rust 命令 | 说明 |
|---|---|---|
| `/api/chat` | `chat` | 调用 LLM API，返回完整回复 |
| `/api/agent` | `agent` | Agent 循环，通过 `app.emit()` 推送事件 |
| `/api/open` | `open_file` | 打开文件/文件夹 |

关键 Rust 依赖：
- `reqwest`：HTTP 请求（调用 DeepSeek API）
- `tokio`：异步运行时
- `tauri::Emitter`：向前端推送事件

### 5.4 前端迁移

- `fetch('/api/...')` → `invoke('command_name')`
- SSE 流 → Tauri 事件监听（`listen('agent-event', ...)`）
- 引入 `@tauri-apps/api` 包

### 5.5 静态导出

- 切换到 `@sveltejs/adapter-static`
- 添加 `+layout.js`：`export const prerender = true; export const ssr = false;`
- `npm run build` 输出到 `build/` 目录

### 5.6 打包结果

```
src-tauri/target/release/conversation-canvas.exe  (13.6MB)
```

- 不需要 Node.js 运行时
- 双击即用
- 前端静态文件内嵌在 exe 中

## Bug 修复记录

| Bug | 原因 | 修复 |
|-----|------|------|
| LiteGraph 内置按钮文字偏下 | `textBaseline` 和 `+1` 偏移 | 改用 Canvas 自定义绘制按钮 |
| Agent 节点间距太近 | `lastY` 从 promptNode 位置开始算 | 改为从 responseNode 底部开始 |
| AI 回复和 Agent 节点重叠 | 未考虑回复节点高度 | `lastY` 初始化为 `responseNode.pos[1] + responseNode.size[1]` |
| Tauri 编译失败 `Emitter` | Rust 2021 edition 不自动导入 trait | 添加 `use tauri::Emitter` |
| Tauri 打包 MSI 失败 | WiX 工具下载网络断开 | 暂用 .exe 分发 |
| Rust 安装慢 | 未设置代理 | 添加 `http_proxy` 环境变量 |

## 技术债务

| 项目 | 说明 | 影响 |
|------|------|------|
| LiteGraph.js 停更 | 最后更新 2023，npm 原版功能有限 | 长期维护风险 |
| API Key 明文存储 | 存在前端内存中 | 安全风险，需系统钥匙串 |
| 无 MSI 安装包 | WiX 下载失败 | 分发不够用户友好 |
| 无自动更新 | 用户需手动下载新版本 | 体验不佳 |
| 无 TypeScript | 全部 JS，类型不安全 | 维护成本高 |
