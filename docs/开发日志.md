## 第一阶段：项目规划与初始化

### 1.1 需求来源

用户提出将 AI 对话系统做成类似 ComfyUI 的节点图画布效果。核心流程：

```
[模型配置卡片] → [输入卡片] → [发送] → [回答卡片] → [新输入卡片] → ...
```

### 1.2 技术选型决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 前端框架 | SvelteKit | 前后端一体，开发效率高 |
| 节点编辑器 | LiteGraph.js（npm 原版） | ComfyUI 同款引擎 |
| 后端 | SvelteKit API Routes | 无需额外服务 |
| AI API | OpenAI 兼容格式 | DeepSeek V4 原生支持 |

### 1.3 项目初始化

- 手动创建 SvelteKit 项目结构（交互式 CLI 在非交互环境下不可用）
- `npm install` 安装依赖：`svelte`, `@sveltejs/kit`, `litegraph.js` 等
- 从 `node_modules/litegraph.js/` 复制 CSS 和 JS 到 `static/litegraph/`
- 配置 `+page.js` 禁用 SSR（LiteGraph 依赖浏览器 API）

### 1.4 初始文件结构

创建了 14 个源文件，覆盖前端画布、节点定义、后端 API、全局样式。

## 第二阶段：核心功能开发

### 2.1 LiteGraph 画布集成

**问题 1：Canvas 尺寸**

LiteGraph 要求 canvas 元素有实际像素宽高，不能只靠 CSS `width:100%`。否则鼠标事件坐标错位，右键菜单和滚轮都不工作。

```javascript
const rect = canvasElement.getBoundingClientRect();
canvasElement.width = rect.width;
canvasElement.height = rect.height;
canvas.resize();
```

**问题 2：npm 原版 vs ComfyUI fork**

最初使用 `addDOMWidget` 添加 HTML 元素到节点中，运行时报错 `this.addDOMWidget is not a function`。

排查发现：npm 上的 `litegraph.js@0.7.18` 是原版，没有 `addDOMWidget`——那是 ComfyUI 自己的 fork 才有的方法。

解决方案：改用原版的 `addWidget` API（支持 `combo`、`string`、`button` 类型）。

### 2.2 自定义节点开发

**三种节点类型：**

| 节点 | 输入 | 输出 | 功能 |
|------|------|------|------|
| 模型配置 | 无 | 配置(object) | 选择模型、API Key、Base URL、System Prompt |
| 对话输入 | 配置(object), 上下文(string) | 消息(string), 配置(object) | 输入消息、发送 |
| AI 回复 | 回复(string) | 内容(string) | 流式显示回复 |

**关键 Bug：LiteGraph 属性名冲突**

LiteGraph 会将 widget 相关属性名注册到节点实例上。最初使用 `_onSend` 作为回调名，被 LiteGraph 覆盖为 widget 对象。解决：改名为 `sendCallback`。

### 2.3 后端 API

- `POST /api/chat` — 接收 model、apiKey、baseUrl、systemPrompt、message、history
- 转发请求到 AI 服务（`baseUrl/chat/completions`）
- SSE 流式返回，前端逐字渲染

### 2.4 对话历史回溯（分叉核心）

不再使用全局平铺数组。每次发送时，从当前 PromptNode 沿图路径向上回溯：

```javascript
function getAncestorPath(node) {
  // 沿 input[1]（上下文）回溯，没有则沿 input[0]（配置）回溯
  // 收集所有祖先节点，reverse 得到从根到当前的路径
}
```

分叉场景下各分支独立回溯，互不干扰。

## 第三阶段：样式与交互优化

### 3.1 深色主题

- 全局 CSS 变量系统（`--bg-canvas`, `--accent`, `--green`, `--amber`）
- LiteGraph 节点颜色覆盖（标题、背景、边框、连线）
- 节点悬停发光效果
- 毛玻璃工具栏

### 3.2 回复节点多行渲染

原版 LiteGraph 的 `string` widget 只支持单行。使用 `addCustomWidget` 自定义绘制：

- Canvas 上逐字符换行绘制多行文本
- 根据内容自动扩展节点高度（80~400px）
- 流式输出时闪烁光标动画
- 鼠标滚轮滚动内容（替代画布缩放）

**滚轮处理的踩坑历程：**

1. 第一次尝试：canvas 上添加 `wheel` 事件监听器 → LiteGraph 用的是 `mousewheel` 事件，不是 `wheel`，不生效
2. 第二次尝试：覆盖 `canvas.processMouseWheel` 方法 → LiteGraph 在初始化时已经 `bind` 了原始方法到监听器，覆盖属性无效
3. 第三次尝试：捕获阶段 + `stopImmediatePropagation` → 同元素同事件类型有效，但跨事件类型无效
4. 最终方案：`removeEventListener` 移除 LiteGraph 的原始监听器，替换为我们的版本，内部手动调用原始缩放逻辑

### 3.3 文本复制

Canvas 绘制的文字无法直接选中复制。方案：点击「选择复制」按钮弹出 DOM textarea 浮层，内容自动全选。

### 3.4 System Prompt

在模型配置节点新增 System Prompt 输入框，后端将其作为 `system` 角色消息插入 messages 首位。

## 第四阶段：自主开发高级功能

### 4.1 画布缩略图（Minimap）

LiteGraph 0.7.18 没有内置 minimap。自行实现：

- 使用 `canvas.onDrawForeground` 回调绘制
- 右下角 180×120px 缩略视图
- 不同类型节点用不同颜色标识
- 显示当前视口范围（红色半透明矩形）
- 支持点击跳转

**Bug：缩略图跟随画布移动**

`onDrawForeground` 在图坐标系中绘制，画布平移时缩略图也跟着动。

修复：绘制前重置坐标系为屏幕坐标：
```javascript
ctx.save();
ctx.setTransform(1, 0, 0, 1, 0, 0); // 重置为屏幕坐标
drawMinimap(ctx);
ctx.restore();
```

### 4.2 节点搜索

工具栏添加搜索框，实时搜索节点中的对话内容（用户消息、AI 回复），定位并高亮匹配节点。

- 支持 Enter/Shift+Enter 上下翻页
- ESC 清空搜索
- 自动平移画布到匹配节点

### 4.3 对比视图

**迭代历程（5 次重构）：**

| 版本 | 方案 | 问题 |
|------|------|------|
| v1 | Ctrl+Click 选中 2 个节点触发 | LiteGraph 单击清空选中，Ctrl 多选不可靠 |
| v2 | `onNodeSelected`/`onNodeDeselected` 回调跟踪 | 回调在某些场景不触发 |
| v3 | 独立 Set 跟踪选中 | HMR 重置 Set，状态丢失 |
| v4 | 弹窗选择模式 | 用户操作步骤多 |
| v5（最终） | 智能触发：2+选中直接对比，0~1弹窗选择 | `node.is_selected` 实时读取 |

**最终方案的关键发现：**

`canvas.selected_nodes` 在某些时机不准确（松开 Ctrl 后被清空）。改用直接遍历所有节点的 `is_selected` 属性，这是 LiteGraph 内部选中状态的源头。

**工具栏点击穿透问题：**

点击工具栏按钮时，mousedown 事件穿透到 canvas，导致 LiteGraph 清空选中。修复：
- 工具栏加 `onmousedown={(e) => e.stopPropagation()}`
- 按钮加 `onmousedown={(e) => e.preventDefault()}`

### 4.4 多模型支持

新增 7 个 AI 模型预设：GPT-4o、GPT-4o Mini、Moonshot v1、通义千问 Max/Plus、GLM-4 Plus、自定义模型。全部使用 OpenAI 兼容格式。

### 4.5 数据持久化

- **导出**：`graph.serialize()` + 额外保存自定义数据 → JSON 文件下载
- **导入**：读取 JSON → `graph.configure()` → 恢复自定义数据

## 关键经验总结

1. **npm 包 vs fork**：使用开源库时要确认 npm 版本和实际使用的 fork 版本功能差异
2. **Canvas 事件系统**：canvas 元素需要显式设置像素尺寸；不同事件类型（wheel/mousewheel/DOMMouseScroll）需要分别处理
3. **第三方库集成**：不要假设回调 API 存在，先 grep 源码确认；bind 过的方法无法通过覆盖属性替换
4. **状态管理**：第三方库的内部状态（如 selected_nodes）可能在非预期时机被清空，优先读取源头属性（is_selected）
5. **事件传播**：浮层 UI 的点击可能穿透到底层 canvas，需要 stopPropagation + preventDefault
