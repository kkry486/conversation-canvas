## 详细实现

### 1. Agent 工具模块

**文件**：`src/lib/tools/index.js`

实现 5 个工具：

| 工具 | 功能 | 参数 |
|------|------|------|
| `read_file` | 读取文件内容 | path |
| `write_file` | 创建/覆盖写入文件 | path, content |
| `list_directory` | 列出目录内容 | path |
| `execute_command` | 执行 shell 命令 | command |
| `search_in_files` | 搜索文件内容 | query, path |

Agent 循环引擎 `runAgentLoop()`：
- 调用 LLM → 检查 tool_calls → 执行工具 → 结果加入消息历史 → 再次调用 LLM
- 最多 10 轮迭代
- 每一步通过 `sendEvent()` 推送前端

### 2. Agent API 路由

**文件**：`src/routes/api/agent/+server.js`

- 接收与 `/api/chat` 相同格式的请求
- 使用非流式调用（`stream: false`），逐轮解析 tool_calls
- SSE 推送 6 种事件：thinking、text_delta、tool_call、tool_result、done、error

### 3. Agent 节点类型

| 节点 | 文件 | 绘制方式 |
|------|------|----------|
| Agent/思考 | `ThinkingNode.js` | addCustomWidget + drawScrollableText |
| Agent/工具调用 | `ToolCallNode.js` | addCustomWidget + drawScrollableText |
| Agent/工具结果 | `ToolResultNode.js` | addCustomWidget + drawScrollableText |

### 4. 文件产物节点

**文件**：`FileProductNode.js`

- 完全自定义绘制（`onDrawForeground`），不使用 LiteGraph 内置按钮
- 自定义按钮绘制：`roundRect` + `textBaseline: 'middle'` 精确居中
- 鼠标悬停高亮：`onMouseMove` 检测按钮区域
- 鼠标点击：`onMouseDown` 检测按钮区域，调用 Tauri invoke

### 5. Tauri 启动器

**架构**：

```
前端（静态 HTML/CSS/JS）← Tauri IPC → Rust 后端（reqwest + tokio）
```

**Rust 后端命令**：

| 命令 | 功能 | 对应原 API |
|------|------|-----------|
| `chat` | 调用 LLM API，返回完整回复 | `/api/chat` |
| `agent` | Agent 循环，通过 `app.emit()` 推送事件 | `/api/agent` |
| `open_file` | 打开文件/文件夹 | `/api/open` |
| `set_work_dir` | 设置工作目录 | 新增 |

**静态导出**：
- `@sveltejs/adapter-static` 输出到 `build/`
- Tauri 配置 `frontendDist: "../build"`
- 前端所有 `fetch('/api/...')` 替换为 `invoke('command_name')`

### 6. Agent 事件流

```
用户输入
    ↓
[对话输入] → [💬 AI回复: 生成中...]
    ↓
Rust 后端调用 LLM
    ↓
LLM 返回 text_delta → 追加到回复节点
    ↓
LLM 返回 tool_call → 创建 [🔧 工具调用] 节点
    ↓
Rust 执行工具 → 创建 [📋 工具结果] 节点
    ↓
如果是 write_file → 额外创建 [📄 文件产物] 节点
    ↓
继续循环或完成 → [💬 AI回复] 状态变为 ✅ 完成
```
