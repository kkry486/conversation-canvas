## 详细实现说明

### 1. 画布缩略图

**实现方式**：在 `init.js` 中使用 `canvas.onDrawForeground` 回调在主画布右下角绘制 minimap。

**功能**：
- 右下角显示所有节点的缩略视图（180×120px）
- 不同类型节点用不同颜色标识（配置=红、输入=绿、回复=黄）
- 显示节点间连线
- 红色半透明矩形标示当前视口范围
- 支持点击 minimap 跳转到对应位置

**技术要点**：
- 计算所有节点的包围盒，缩放到 minimap 尺寸
- 视口位置通过 `canvas.ds.offset` 和 `canvas.ds.scale` 计算
- 点击事件通过坐标反算定位到图坐标

### 2. 节点搜索

**实现方式**：在工具栏添加搜索框（`+page.svelte`），搜索节点中的对话内容。

**功能**：
- 实时搜索：输入即搜索，无需回车
- 搜索范围：节点的 `_promptText`（用户消息）、`_aiResponse`（AI 回复）、`_responseText`（回复显示）
- 匹配计数：显示 "第X/共Y个"
- 上下翻页：‹ › 按钮或 Enter/Shift+Enter
- 定位功能：自动平移画布到匹配节点并选中高亮
- ESC 清空搜索

### 3. 对比视图

**实现方式**：检测选中的回复节点，满足条件时在工具栏显示对比按钮。

**功能**：
- Ctrl+Click 选中 2 个回复节点后，工具栏出现「⚡ 对比分支」按钮
- 点击按钮弹出全屏对比弹窗
- 左右并排显示两个节点的回复内容
- 每列独立滚动
- 点击遮罩层或 ✕ 关闭

**触发条件**：`canvas.selected_nodes` 中恰好有 2 个 `Chat/回复` 类型节点

### 4. 多模型支持

**实现方式**：在 `config.js` 的 `MODEL_PRESETS` 数组中新增模型预设。

**新增模型**：

| 模型 | 提供商 | Base URL |
|------|--------|----------|
| GPT-4o | OpenAI | `https://api.openai.com/v1` |
| GPT-4o Mini | OpenAI | `https://api.openai.com/v1` |
| Moonshot v1 128K | 月之暗面 | `https://api.moonshot.cn/v1` |
| 通义千问 Max | 阿里云 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| 通义千问 Plus | 阿里云 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| GLM-4 Plus | 智谱 | `https://open.bigmodel.cn/api/paas/v4` |
| 自定义模型 | 用户自定义 | 手动填写 |

所有模型均使用 OpenAI 兼容的 `/chat/completions` 接口格式。

### 5. 数据持久化

**实现方式**：在 `init.js` 中实现 `exportGraph` 和 `importGraph` 函数。

**保存功能（exportGraph）**：
- 调用 LiteGraph 内置的 `graph.serialize()` 序列化图结构
- 额外保存每个节点的自定义数据（对话内容、状态、配置等）
- 生成 JSON 文件，文件名包含日期
- 通过 Blob + `<a>` 标签触发浏览器下载

**加载功能（importGraph）**：
- 读取 JSON 文件，调用 `graph.configure()` 恢复图结构
- 从 `_customData` 恢复节点的自定义数据（对话文本、AI 回复等）
- 自动重绘画布

**工具栏按钮**：
- 💾 保存按钮：导出当前对话图为 JSON
- 📂 加载按钮：从 JSON 文件导入对话图
